# NewLife.NovaDb 存储架构

> 对应模块：S01-S06（Storage 领域）
> 关联模块：L01-L05（WAL/Binlog）、N02/N05（索引冷热分离 → 见[索引架构.md](索引架构.md)）

---

## 1. 设计目标

| 目标 | 说明 |
|------|------|
| 文件夹即数据库 | 一个目录 = 一个数据库，拷贝即迁移 |
| MMF 驱动 | 64 位虚拟地址空间 128TB，OS 按需映射 |
| 每表独立文件组 | 表间 IO 与锁完全隔离 |
| 纯托管 | 无 Native 依赖，全 C# 实现 |

## 2. 数据库目录结构

### 2.1 文件分类

**数据库级文件**：

| 文件 | 命名规则 | 说明 |
|------|---------|------|
| 元数据 | `nova.db` | 表/列/索引定义，二进制格式 |
| Binlog | `nova.{SeqNo:D6}.binlog` | 逻辑日志（按文件滚动） |

**表级文件组**：

| 后缀 | 说明 |
|------|------|
| `.nvd` | 行数据（MMF 访问） |
| `.nvi` | 主键索引 |
| `_{IndexName}.nvi` | 二级索引（每索引一个文件） |
| `.wal` | 写入时产生，checkpoint 后删除 |
| `.lock` | 嵌入模式表级写锁 |

### 2.2 文件命名

| 文件类型 | Nova 引擎 | Flux 引擎（时序分片） |
|---------|----------|---------------------|
| 数据文件 | `{Table}.nvd` | `{Table}.nvd`（当前）/ `{Table}_{yyyyMMdd}.nvd`（按天）/ `{Table}_{yyyyMMddHH}.nvd`（按小时） |
| 主键索引 | `{Table}.nvi` | `{Table}.nvi` |
| 二级索引 | `{Table}_{Idx}.nvi` | `{Table}_{Idx}.nvi` |

### 2.3 目录示例

```
StarDust/                              ← 文件夹即数据库
├── nova.db                            ← 元数据
├── nova.000001.binlog                 ← Binlog
├── nova.000002.binlog
│
├── User.nvd / User.nvi / User_Name.nvi / User.wal / User.lock   ← Nova 表
├── App.nvd / App.nvi / App_Name.nvi
│
├── AppLog.nvd / AppLog.nvi            ← Flux 时序表（当前分片）
├── AppLog_20250101.nvd                ← 历史分片
├── AppLog_20250102.nvd
│
├── SessionCache.nvd / SessionCache.nvi   ← KV 表
└── ConfigCache.nvd / ConfigCache.nvi
```

## 3. 文件格式

### 3.1 文件头（FileHeader，32 字节）

所有文件共享统一文件头：

```
偏移  大小  字段            说明
0     4B   Magic           0x4E4F5641 ("NOVA")
4     1B   Version         格式版本（当前 1）
5     1B   FileType        1=Data 2=Index 3=Wal 4=Binlog 5=Metadata
6     1B   PageSizeShift   页大小 = 1 << shift
7     1B   Flags           bit0=加密 bit1=压缩 bit2=只读
8     8B   CreateTime      UTC 毫秒
16    12B  Reserved        填零
28    4B   Checksum        CRC32（覆盖 0-27）
```

校验：读取时重新计算 CRC32 并比对，不一致则文件损坏。

### 3.2 元数据文件（nova.db）

```
nova.db:
├── FileHeader (32B, FileType=Metadata)
├── DatabaseInfo { Name, CreateTime, ModifyTime }
└── TableCatalog[]
    ├── TableName, EngineType(0=Nova 1=Flux 2=KV), Comment, CreateTime
    ├── Columns[] { Name, DataType, Length, IsNullable, IsPrimaryKey, IsAutoIncrement, DefaultValue, Comment }
    └── Indexes[] { Name, IsUnique, ColumnNames[] }
```

**修改机制**：先写 `nova.db.tmp`，完成后原子重命名，保证崩溃安全。

## 4. 页面管理

### 4.1 页面布局

```
┌────────────────────────────┐
│     FileHeader (32B)        │
├────────────────────────────┤
│  Page 0 = PageHeader + Data │  offset = 32 + 0 × PageSize
│  Page 1 = PageHeader + Data │  offset = 32 + 1 × PageSize
│  ...                        │
└────────────────────────────┘
```

### 4.2 页头（PageHeader，32 字节）

```
偏移  大小  字段         说明
0     8B   PageId       全局唯一
8     1B   PageType     0=Empty 1=Data 2=Index 3=Directory 4=Metadata
9     1B   Flags        bit0=加密 bit1=压缩 bit2=脏页
10    2B   Reserved
12    8B   LSN          日志序列号（WAL 恢复比对）
20    4B   DataLength   有效数据长度
24    4B   Reserved
28    4B   Checksum     CRC32
```

### 4.3 行存储

数据页内行按追加排列：

```
PageHeader (32B)
Row 0: [RowLen(4B)] [Col0] [Col1] ...
Row 1: [RowLen(4B)] [Col0] [Col1] ...
(空闲空间)
```

列编码规则由 `IDataCodec` 定义。NULL 用长度前缀 -1 表示。

## 5. MMF 分页访问器（MmfPager）

利用 `MemoryMappedFile`，64 位系统 OS 仅映射实际访问的页到物理内存。

```
物理内存（有限）            虚拟地址空间（128TB）
┌──────────┐             ┌─────────────────┐
│ 热点页面  │ ←─ 映射 ── │  Page 3 (热)    │
│ Page 3   │             │  Page 7 (热)    │
│ Page 7   │             │  Page 12 (热)   │
└──────────┘             │  Page N ← OS 按需加载
                         └─────────────────┘
```

核心能力：

- 自动写入/验证 FileHeader
- `ReadPage(pageId)` / `WritePage(pageId, data)` 随机读写
- 可选 CRC32 校验
- 写入超范围时自动扩展文件
- 线程安全（内部锁保护）

配置：`PageSize`=4096、`EnableChecksum`=true、`PageCacheSize`=1024。

---

## 6. 页面缓存（PageCache）

基于 `LinkedList + Dictionary` 的 O(1) LRU：

```
最近访问 ← [Page7] ←→ [Page3] ←→ [Page12] → 最久未访问（淘汰）
```

- **TryGet**：命中移至头部
- **Put**：满时淘汰尾部，新页插入头部
- **监控**：HitCount / MissCount / Count / Capacity

---

## 7. 并发控制

| 模式 | 引擎并发 | 跨进程锁 | 说明 |
|------|---------|---------|------|
| 嵌入 | MVCC（进程内） | 表级文件锁 `.lock` | 一写多读，表间互不影响 |
| 服务器 | MVCC | 无（单进程） | SqlEngine 统一调度 |
| 只读 | 只读 | 无 | 多进程可同时读 |

表级文件锁：0 字节文件，利用 `FileStream` 排他锁实现互斥，粒度优于 SQLite 的库级锁。

## 8. 存储配置汇总

### 8.1 DbOptions（嵌入 + 服务器共用）

| 配置项 | 默认值 | 说明 |
|--------|-------|------|
| Path | — | 数据库目录 |
| WalMode | Normal | Full/Normal/None |
| PageSize | 4096 | 页大小（字节） |
| PageCacheSize | 1024 | LRU 缓存页数 |
| HotWindowSeconds | 600 | 热数据窗口（秒） |
| ColdEvictionSeconds | 1800 | 冷数据淘汰（秒） |
| EnableChecksum | true | CRC32 校验 |
| ReadOnly | false | 只读模式 |

### 8.2 ServerDbOptions（服务器附加）

| 配置项 | 默认值 | 说明 |
|--------|-------|------|
| EnableBinlog | true | 启用 Binlog |
| BinlogMaxFileSize | 256MB | 单文件滚动阈值 |
| BinlogRetentionDays | 0 | 保留天数 |
| BinlogMaxTotalSize | 0 | 总大小上限 |

## 9. CRUD 数据流（存储视角）

### 写入

```
Insert → 序列化(IDataCodec) → WAL 追加 → SkipList 插入 → MmfPager.WritePage → PageCache 更新 → Binlog(服务器模式)
```

### 读取

```
Get → 热索引(SkipList) → 未命中 → 冷索引(稀疏目录) → 锚点定位 → .nvi 加载 → PageCache → MmfPager → MVCC 可见性 → 反序列化
```

### Checkpoint

```
后台线程 → 扫描 WAL UpdatePage → 脏页刷新到 .nvd → 删除/截断 .wal
```

## 10. 设计决策

| # | 决策 | 要点 |
|---|------|------|
| D1 | 纯托管实现 | 无 native 依赖，跨平台一致 |
| D2 | MMF 存储 | 利用 OS 虚拟内存，128TB 地址空间 |
| D3 | 表级 WAL | 独立 WAL 减少竞争，简化恢复 |
| D4 | Nova 不分片 | 单表单文件，MMF 高效处理超大文件 |
| D5 | nova.db 元数据 | 独立元数据，DDL 原子写入 |
| D6 | Binlog 独立于 WAL | 职责正交，互不依赖 |
| D7 | Binlog 可手工删除 | 文件自包含，删除不导致崩溃 |
