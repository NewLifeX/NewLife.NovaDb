# NewLife.NovaDb

以 **C#** 实现、运行于 **.NET 8** 平台的中大型混合数据库：嵌入式/服务器双模，融合关系型、时序、消息队列、NoSQL(KV) 能力。

## 产品介绍

`NewLife.NovaDb`（简称 `Nova`）是面向 .NET 应用的一体化数据基础设施，目标是在裁剪大量冷门特性的前提下，获得更高读写性能与更低运维成本，并支持逻辑上无上限的数据量（受磁盘与切分策略约束）：

- **双部署模式**：
  - **嵌入模式**：像 SQLite 一样以库的形式运行，数据存储在本地文件夹
  - **服务器模式**：独立进程 + TCP 协议，像 MySQL 一样网络访问；支持集群部署与主从同步（一主多从）
- **文件夹即数据库**：拷贝文件夹即可完成迁移/备份，每表独立文件组（例如：`.data`/`.idx`/`.wal`）。
- **关系型 + 时序 + Stream/MQ + KV**：
  - 通用关系型引擎（面向 CRUD 与事务）
  - 时序/追加写引擎（适用于 IoT/日志等场景）
  - 参考 Redis Stream 的消费组 + Pending + Ack 语义的 Stream/MQ
  - KV 作为关系型表的逻辑视图；每行 `Key + Value + TTL`，支持分布式部署
- **纯托管代码**：不依赖 Native 组件，便于在多平台与受限环境部署。

## 规划能力（Roadmap）

- **Join 优化**：初期以 Nested Loop 为主，后续支持 Hash Join。
- **时序压缩**：支持对冷数据（例如“今天以前”的分片）进行压缩存储，用 IO 换 CPU；压缩查询可能增加解压开销。
- **时序降采样**：支持按时间窗口降采样/聚合（优先级较低）。
- **集群与主从同步**：独立服务支持集群与一主多从同步；MQ 模式同样支持集群部署。

## 数据安全与 WAL 模式

NovaDb 提供三种 WAL 持久化策略：

- `FULL`：同步落盘（更强的数据安全）
- `NORMAL`：异步 1s 刷盘（默认）
- `NONE`：全异步（最高吞吐）

只要不选择同步模式（`FULL`），就意味着接受在崩溃/断电等场景下可能发生数据丢失。
